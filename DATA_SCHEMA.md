# DATA_SCHEMA.md

Специфікація для `data/drives.json` — єдиного джерела правди цього проєкту.

## Розташування файла

`data/drives.json` у корені цього репозиторію.

## Верхньорівнева структура

```ts
{
  updatedAt: string | null,   // ISO 8601 UTC-мітка часу найсвіжішого сканування
                              //   серед УСІХ дисків. Оновлюється сканером.
                              //   null, якщо сканувань ще не було.
  drives: Drive[]             // Масив записів дисків. Може бути порожнім.
}
```

Один запуск сканера оновлює **лише обрані** диски (upsert за `name`). Записи дисків, які не сканували в цьому запуску, залишаються без змін.

## Об'єкт `Drive`

```ts
{
  name: string,               // Унікальний ключ. Зберігається точно як увів користувач.
                              //   Приклади: "Black 3", "Gray 1"
                              //   Без slugify, без нормалізації регістру.

  scannedAt: string,          // ISO 8601 UTC. Коли востаннє сканувався САМЕ цей диск.

  filesystem: string,         // Тип файлової системи, який повертає ОС.
                              //   Приклади: "APFS", "HFS+", "exFAT", "NTFS",
                              //   "MS-DOS FAT32", "Mac OS Extended"

  totalBytes: number,         // Повний обсяг тома у байтах (ціле число).
  freeBytes: number,          // Вільний простір у байтах (ціле число).
  usedBytes: number,          // Використаний простір у байтах (ціле число).
                              //   Примітка: usedBytes може не дорівнювати точно
                              //   totalBytes - freeBytes через накладні витрати
                              //   файлової системи. Записувати те, що дає ОС.

  entries: Entry[]            // Записи кореневого рівня диска.
                             //   Під час запису сортуються як у файловому менеджері:
                             //   спершу папки, потім файли; у межах групи — за name.
}
```

## Об'єкт `Entry` (папка)

```ts
{
  type: "folder",
  name: string,               // Назва папки (basename). UTF-8.
  sizeBytes: number           // Сумарний рекурсивний розмір у байтах.
}
```

## Об'єкт `Entry` (файл)

```ts
{
  type: "file",
  name: string,               // Повна назва файла з розширенням. UTF-8.
  ext: string,                // Розширення у нижньому регістрі без початкової крапки.
                              //   Поле повністю пропускається для файлів
                              //   без розширення (не ставити "" чи null).
  sizeBytes: number
}
```

## Правила

### Режим доступу (безпека)

- Сканери читають лише metadata і розміри з підключеного тому.
- Запис/видалення/перейменування файлів на сканованому диску **заборонені**.
- Єдиний дозволений запис — `PUT` оновленого `data/drives.json` у GitHub Contents API.

### Ідентичність диска

- `name` — первинний ключ. Сканер робить **upsert** за `name`: якщо диск існує в `drives[]`, повністю замінює запис; якщо ні — додає новий.
- Назва диска береться з label/назви тому (поточна реалізація без ручного override у скрипті).
- **Ніколи** не перейменовувати автоматично, не slugify, не переводити в нижній регістр і не модифікувати назви дисків.

### Що виключається з `entries`

- Усі приховані / dotfiles на macOS (усе, що починається з `.`):
  `.DS_Store`, `.Spotlight-V100`, `.fseventsd`, `.Trashes`, `.TemporaryItems`,
  `.DocumentRevisions-V100`, `.apdisk` тощо
- Також не включати службові кореневі папки:
  `System Volume Information`, `$RECYCLE.BIN`, `$Recycle.Bin`, `RECYCLER`
- Системні папки на Windows:
  `System Volume Information`, `$RECYCLE.BIN`, `$Recycle.Bin`, `RECYCLER`
- Файли / папки з атрибутом `Hidden` або `System` на Windows

### Сортування

- Сканери мають сортувати `entries[]` у порядку "як у macOS/Windows file manager":
  1) спочатку `type: "folder"`, потім `type: "file"`;
  2) у межах кожної групи — за `name` за зростанням (case-insensitive).
- Сканери сортують `drives[]` за `name` за зростанням перед записом.

### Розміри

- Усі розміри — у **байтах** як **цілі числа** (JSON `number`, але без дробової частини).
- Для відображення в UI використовувати одиниці на базі 1024 (бінарні): `ТБ`, `ГБ`, `МБ`, `КБ`, `Б`.
- Розмір папки = рекурсивна сума всіх файлів усередині:
  - macOS: `du -sk "$folder" | awk '{print $1}'` × 1024
    (важливо: **обов'язково з `-k`**, бо без нього `du` повертає у блоках 512 байт або у тому, що задано `BLOCKSIZE`; сканер також має `unset BLOCKSIZE` для гарантії).
  - Windows: `(Get-ChildItem -LiteralPath $folder -Recurse -File -Force | Measure-Object Length -Sum).Sum`

### Інформація про том

- macOS: `df -k "$volume"` — другий стовпчик total, третій used, четвертий free (всі в KB → ×1024 для байтів).
  Тип файлової системи: `diskutil info "$volume"` → поле `File System Personality`.
- Windows: `Get-Volume -DriveLetter X` → `.Size`, `.SizeRemaining`, `.FileSystemType`.

### Мітки часу

- ISO 8601 UTC із суфіксом `Z`.
- Формат: `YYYY-MM-DDTHH:MM:SSZ` (точність до секунд, без дробових секунд, без offset).
- Генерується в момент сканування, **не** з file system mtime.
- macOS: `date -u +%Y-%m-%dT%H:%M:%SZ`
- Windows: `(Get-Date).ToUniversalTime().ToString("yyyy-MM-ddTHH:mm:ssZ")`

### Розширення

- Нижній регістр, без початкової крапки.
- Для файлів із кількома крапками (`my.wedding.final.mp4`) використовувати **тільки останній сегмент**: `mp4`.
- Для файлів без розширення (`Makefile`, `README`) поле `ext` **повністю пропускати** (не порожній рядок, не `null`).

### Форматування JSON

- Гарно відформатований із відступом 2 пробіли.
- Порожній рядок у кінці файла (`\n`).
- Кодування UTF-8.
- Без BOM.
- Перед PUT — кодувати в base64 (без переносів рядків): `base64` у macOS, `[Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes(...))` у Windows.

```json
{
  "updatedAt": null,

```

## Валідація

Примусова валідація схеми не використовується. Обидва сканери — довірені джерела запису. Веб-інтерфейс під час читання має працювати захисно:

- Коректно обробляти відсутні поля.
- Якщо у диска відсутнє `entries`, трактувати як `[]`.
- Якщо у файлового запису відсутнє `ext`, для відображення брати fallback із парсингу `name` (`name.split('.').pop()` або порожньо, якщо крапки немає).
- Якщо `updatedAt` дорівнює `null`, у UI приховувати мітку «Оновлено».

## Версія

Поля `schemaVersion` немає. Якщо схему колись доведеться змінити, агент має:
1. Оновити обидва сканери й веб-читач в одному коміті.
2. Додати тут примітку про міграцію.
3. За потреби додати `schemaVersion` саме на цьому етапі.
